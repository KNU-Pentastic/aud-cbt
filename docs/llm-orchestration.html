1

안전 분류 — 가장 먼저, 무조건

규칙 키워드+ Haiku safety_classifier.py

규칙(키워드 스캔) + LLM 분류를 둘 다 돌려 더 높은 위험 등급을 채택합니다. "재현율 > 정밀도" 원칙 — 진짜 위험을 놓치는 것이 거짓 경보보다 훨씬 나쁩니다. (예: "죽고 싶지 않아" 같은 부정형은 오탐을 막도록 별도 처리.)

입력: 환자 발화 + 최근 맥락→ 출력: grade A / B / none + event_type
GRADE A 응급 → 즉시 차단

자살 위험 · 급성 중독. 이 발화를 맥락에서 제외하고 환자를 LLM 잠금 상태로 전환합니다. 119·1393 자원 안내로 넘어가며, 의료진이 풀어줄 때까지 일반 대화는 멈춥니다. 파이프라인은 여기서 종료.

GRADE B 비응급 → 방향 전환

재음주 → 재음주 대응(RESU) 대화로, 복약 중단 → 복약 지원(SOMA) 대화로 컨텍스트를 전환합니다. PAWS(급성후증후군)는 자원 안내. 대화는 계속됩니다.

🛡️ 안전 게이트가 먼저인 이유

  • 규칙 + LLM 하이브리드. 키워드 룰은 빠르고 확실하게, LLM은 완곡어법·맥락까지 — 둘 중 더 높은 등급을 채택.
  • 재현율 우선. 애매하면 더 위험한 쪽으로 분류. "마음이 오히려 편안해요" 같은 위장된 신호도 응급으로.
  • 오탐 루프 방지. 위기 발화는 이후 맥락에서만 도려내, 의료진이 잠금을 푼 뒤 재잠금·고착되지 않게 함 (임상 기록엔 보존).
  • 의료진 잠금/해제. grade A는 llm_locked → 의료진 웹에서만 해제 가능.
  • 관찰 가능성. LLM_TRACE=on이면 참고한 프롬프트·필터 결과·진행도를 SSE로 라이브 노출 (정량 평가용).
  • 비차단 설계(fail-safe). 출력가드·발화분석·요약이 실패해도 대화는 멈추지 않습니다. 출력가드는 고위험만 답변을 대체(fallback)하고, 나머지는 원문을 그대로 통과시킵니다.

용어 한눈에

오케스트레이션
역할이 다른 여러 LLM 호출을 정해진 순서·조건으로 묶어 하나의 결과를 만드는 것.
CBI
NIAAA의 결합 행동 개입(Combined Behavioral Intervention). 이 앱의 치료 콘텐츠 골격.
MI
동기강화상담(Motivational Interviewing). 코치 어조의 기본 원칙(공통 헤더).
Phase / 모듈
12주를 4개 Phase로 나누고, Phase 3는 9개 기술 모듈 중 환자별 1~2개를 선택.
grade A / B
안전 등급. A=응급(즉시 잠금), B=비응급(컨텍스트 전환/자원 안내).
Pull-out
정규 Phase를 잠시 벗어나는 전용 대화(RESU 재음주·SOMA 복약).
SSE
Server-Sent Events. 답변 토큰을 실시간으로 한 글자씩 밀어주는 스트리밍 방식.
출력 가드
생성된 답변을 환자에게 보내기 전 사후 검수하는 안전 필터.
LLM_TRACE
관찰/평가용 스위치. 참고 프롬프트·필터 결과·진행도를 SSE로 함께 노출(운영에선 off 권장).
기술 스택   FastAPI · SSE(sse-starlette) · SQLAlchemy · PostgreSQL · Redis · Anthropic Python SDK · JWT(HS256) · 환자앱(RN/Expo) + 의료진 웹
AUD-CBT 백엔드 LLM 오케스트레이션 시각화 · 코드 기준(backend/app/services/*, prompts/cbi/v3)으로 작성. 모델 id는 config.py 설정값(claude-haiku-4-5 / sonnet-4-6 / opus-4-7) 기준이며 운영 설정에 따라 달라질 수 있습니다. ＊ 키 미설정 시(기본값 USE_LLM_MOCK=true)엔 실제 Anthropic 호출 없이 결정적 mock으로 동작합니다.
